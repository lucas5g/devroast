# Specs

## Formato

Cada spec deve ser um arquivo `.md` em `@specs/` com o nome: `<feature>-spec.md`

## Estrutura

```markdown
# <Feature Name>

## Problema
<Breve descrição do problema que esta feature resolve>

## Solução
<Como a feature resolve o problema>

## Escopo
- Escopo: frontend/backend/etc
- Dependências: <se houver>

## Critérios de aceite
1. <Critério 1>
2. <Critério 2>
```

## Processo
1. Criar spec antes de implementar
2. Revisar com o time
3. Implementar seguindo o spec
4. Validar critérios de aceite
